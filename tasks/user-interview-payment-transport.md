## 背景・きっかけ

- オークション請求書をアップロードしても、支払い期日がどこにも自動で記録されない
- USSやLAAは請求書に支払日が書いていないので、自分で計算しないといけない
- 「今日アップした時に、どこにも支払い日と書いてある場所がない」

## アーキテクチャ決定

### データソース
- **車両管理・陸送依頼書**：Vehicle table（HITL後）→ 車体番号の精度を確保するため
- **支払いカレンダー**：InvoiceJobs table（請求書アップ後）→ 合計金額は請求書レベルのデータのため

### なぜ車両合計の計算ではダメか（実例）
- HAA請求書：前回繰越¥(358,690)が含まれる
- TAA請求書：前回残高¥2,271,460 + 当日入金 + 陸送料・再出品料が混在 → 本日差引合計¥53,900
- 車両金額を足し算しても実際の支払い金額と一致しない
- 請求書に印字された「本日差引合計」「今回合計」を直接抽出するのが正解

### 請求書アップの運用変更
- 現状：車両のある請求書だけアップ（Vehicle tableに入れるため）
- 新運用：**全請求書をアップする**（陸送料・再出品料のみの請求書も含む）
- 理由：支払いカレンダーには全ての請求書の合計が必要

### 請求書レベル vs 車両レベルの分離（重要な設計変更）

現状の問題：
- `auction`・`auctionDate`が車両ごとに重複保存されている
- 9台の請求書で開催日を1つ修正しようとすると9回修正が必要
- `invoice_total`は車両レベルに収まらない（請求書1枚に1つ）

**決定：請求書レベルと車両レベルを分離する**

#### InvoiceJobs.Json の構造変更
```js
// 現状（車両レベルに重複）
{ items: [
  { auction: "HAA神戸", auction_date: "2026/01/24", chassis: "DA63T", charges: [...] },
  { auction: "HAA神戸", auction_date: "2026/01/24", chassis: "3B20",  charges: [...] },
]}

// 新設計（請求書レベルに分離）
{
  auction: "HAA神戸",          // ← トップレベルに移動
  auction_date: "2026/01/24",  // ← トップレベルに移動
  session: "第1285回",         // ← 新規追加（開催回）
  invoice_total: 718190,       // ← 新規追加（請求書合計）
  items: [
    { chassis: "DA63T", charges: [...] }, // auctionなし
    { chassis: "3B20",  charges: [...] },
  ]
}
```

#### DBへの反映
- `InvoiceJobs`：`invoiceTotal Int?` を追加（最終ページのレコードに保存）
- `Vehicle table`：変更なし（保存時に全車両に一括コピー）
- `paymentDueDate`：DBに保存せず、カレンダー表示時にルールで計算

#### HITL UIの変更
```
┌─────────────────────────────────────────┐
│ 会場：HAA神戸   開催日：2026/01/24       │ ← 請求書レベル（1回編集で全台に反映）
│ 開催回：第1285回   合計：¥718,190        │
├─────────────────────────────────────────┤
│ DA63T 518996   95%   車両レベル          │
│  落札金額 ¥148,000 / リサイクル ¥3,080  │
├─────────────────────────────────────────┤
│ 3B20 503006    88%                      │
└─────────────────────────────────────────┘
```

### 実装順序
1. **AIスキーマ変更**：`auction`・`auction_date`・`session`・`invoice_total`をトップレベルに移動
2. **InvoiceJobs schema**：`invoiceTotal Int?` を追加
3. **HITL UI**：請求書ヘッダーセクション追加
4. **保存API**：トップレベルから取得して全車両にコピー
5. **支払いカレンダーUI**：InvoiceJobsから読み込んで表示
